# REST API Design

## When to Use

Use REST quando recursos, colecoes e estados podem ser expostos por URLs estaveis, metodos HTTP e representacoes previsiveis.

## What Breaks First

Endpoints viram comandos soltos, status codes perdem significado, paginacao e filtros mudam sem contrato, e erro nao permite recuperacao.

## Interview Trap

Discutir REST purista e esquecer ergonomia. O cliente precisa saber como criar, ler, alterar, paginar, filtrar, versionar e lidar com erro.

## Practice Drill

Desenhe API de pedidos com listagem paginada, detalhe, criacao idempotente, cancelamento e resposta de erro padronizada.
